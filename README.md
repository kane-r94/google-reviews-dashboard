## Folder structure

```
google-reviews-dashboard/
├── .github/
│   └── workflows/
│       ├── update_reviews.yml       ← runs every Monday — updates ratings in the dashboard HTML
│       └── update_review_text.yml   ← runs every day — fetches all review text
├── credentials/                     ← NEVER commit this folder's contents
│   ├── oauth_client.json            ← already done
│   └── token.json                   ← auto-created on first run
├── fetch_reviews.py                 ← fetches ratings, updates dashboard HTML
├── fetch_review_text.py             ← fetches all review text, saves reviews_data.json
├── list_locations.py                ← run once to find your location IDs
├── config.json                      ← property names + location IDs (to fill in)
├── requirements.txt
├── google_reviews_dashboard.html    ← ratings dashboard (GitHub Pages)
├── reviews.html                     ← filterable reviews feed (GitHub Pages)
├── reviews_data.json                ← auto-generated by fetch_review_text.py
└── .gitignore                       ← keeps credentials out of git
```

## When API approval arrives

Google will email you to confirm access has been granted. Once you receive that email, follow the steps below in order.

### Step 1 — Install dependencies

Open **Command Prompt** (press `Win + R`, type `cmd`, press Enter):

```
cd C:\Users\KaneRochford\google-reviews-dashboard
pip install -r requirements.txt
```

### Step 2 — Find your location IDs

Run the location finder script:

```
python list_locations.py
```

A browser window will open — sign in with the Google account that manages your Business Profiles and click **Allow**. The script will print every property name alongside its full location ID, which looks like:

```
accounts/123456789/locations/987654321
```

### Step 3 — Fill in config.json

Open `config.json` in Notepad and:

1. Replace `YOUR_GITHUB_USERNAME` with your GitHub username
2. Replace `YOUR_REPO_NAME` with the repository name where your dashboard HTML lives
3. Replace each `Property Name N` with your actual property names (matching the names in your existing dashboard)
4. Replace each `LOCATION_ID_N` with the matching location ID from Step 2
5. Optionally add Trustpilot and AllAgents URLs for any properties that have them

### Step 4 — Copy your dashboard HTML into the project

Copy your existing `google_reviews_dashboard.html` into this folder (next to `fetch_reviews.py`).

The script looks for this exact line somewhere in a `<script>` block in the HTML:
```javascript
const properties = [
```
It replaces the entire array with fresh data on each run.

### Step 5 — Run the script for the first time

```
python fetch_reviews.py
```

The script will:
- Sign in to Google (browser opens automatically on first run)
- Fetch ratings for all 26 properties
- Update the HTML file with the new data
- Print a summary to the screen

Check the HTML file afterwards — the `const properties = [` array should now contain your live ratings.

### Step 6 — Push the project to GitHub

Push this entire project folder to your existing GitHub repository (the one that serves your GitHub Pages dashboard).

If you're new to git on Windows, download **GitHub Desktop** from `https://desktop.github.com` — it gives you a graphical interface and handles this without needing to use the command line.

Make sure `.gitignore` is included in the push — it prevents the `credentials/` folder from ever being uploaded.

### Step 7 — Add the Google token as a GitHub Secret

After running the script locally (Step 5), a `credentials/token.json` file will exist. This needs to be added to GitHub as an encrypted secret so the automation can authenticate without you being present.

1. Open PowerShell and run:
   ```powershell
   $bytes = [System.IO.File]::ReadAllBytes("credentials\token.json")
   [Convert]::ToBase64String($bytes) | clip
   ```
   This copies the encoded token to your clipboard.

2. Go to your GitHub repository → **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Name it exactly: `GOOGLE_TOKEN_JSON`
5. Paste from clipboard into the value field
6. Click **Add secret**

### Step 8 — Test the automation

1. Go to your repository on GitHub → **Actions** tab
2. Click **Update Review Data** in the left sidebar
3. Click **Run workflow** → **Run workflow** to trigger a manual test
4. Watch the logs to confirm it completes successfully

From this point the workflow runs automatically every Monday at 08:00 UTC and pushes the updated HTML to GitHub Pages.

## How it works

```
Every Monday 08:00 UTC
        ↓
GitHub Actions runs fetch_reviews.py
        ↓
Script authenticates with Google using saved token
        ↓
Fetches rating + review count for each of 26 properties
        ↓
Scrapes Trustpilot / AllAgents for any properties with URLs set
        ↓
Injects fresh data into google_reviews_dashboard.html
        ↓
Pushes updated HTML to GitHub
        ↓
GitHub Pages publishes it — dashboard is live
```

## Troubleshooting

| Problem | Fix |
|---|---|
| `ERROR: OAuth client file not found` | Check that `oauth_client.json` is in the `credentials/` folder |
| `Could not find 'const properties'` | Check your HTML file contains `const properties = [` exactly |
| API returns N/A for a property | Check the location ID in config.json is correct and the Business Profile is verified |
| Trustpilot / AllAgents shows N/A | Check the URL in config.json points to the correct branch page |
| GitHub Actions fails with 401 | Re-run the script locally, re-encode the token, and update the GitHub secret (see below) |

---

## Refreshing the Google token

Google OAuth tokens expire periodically. If the automation starts failing with an authentication error:

1. Delete `credentials/token.json` on your local machine
2. Run `python fetch_reviews.py` locally — a browser sign-in will be triggered
3. Re-run the PowerShell command from Step 7 to encode the new token
4. Update the `GOOGLE_TOKEN_JSON` secret in GitHub with the new value
